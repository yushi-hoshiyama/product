# DELNE ブランドガイドライン

DELNEのブランド表現を統一するためのガイドライン。
LP・広告・SNS・プレスリリース・営業資料・社外プレゼン等、対外的に出る全てのコンテンツで参照する。

> 📌 製品事実は `projects/delne/product/summary.md` (single source of truth)、表現規範の法令・配慮ルールは `projects/delne/marketing/strategy/compliance.md` を参照。
> 本ガイドは **どう"らしさ"を表現するか** のルールであり、両者と矛盾しないように運用する。
> 矛盾を発見した場合は本ガイドを修正し、根本ルール (CLAUDE.md / summary.md / compliance.md) には触れない (統合企画に上申)。

## 0. このガイドの使い方

1. コンテンツ制作前に **2章 (トーン&マナー) と 3章 (人称・文体)** を読む
2. 制作後、**9章 (チェックリスト)** で自己確認
3. 法令・配慮上の最終判断は `compliance.md` を優先 (本ガイドはその上の表現規範)

## 1. ブランドの精神

### 1.1 私たちが解決すること

認知症のご家族からの頻回電話を、家族の声を再現したAIが24時間365日丁寧に受ける。
「ケアの孤立」を技術と経済の両側面から解消し、社会インフラとして再設計する。

### 1.2 ブランドの本質 (Essence)

> **「でんわにデルネ」 — そばにいられない時間を、やさしく預けられる場所。**

### 1.3 ブランドが大切にする3つの態度

| 態度 | 説明 |
|---|---|
| **やさしさ** | 認知症のある方にも、ご家族にも、結論を急がず・否定しない・責めない |
| **誠実さ** | できることだけを言う。効果を盛らない。検証可能な事実だけを訴求する |
| **共にあること** | 解決を押し付けず、ご家族の選択を支える。介護を「代行」するのではなく「支援する」 |

### 1.4 ブランドのキーワード (Brand Vocabulary)

DELNE らしいキーワード。コピー・コンテンツで意識的に使う。

- **やさしく / 丁寧に / そばに / 寄り添う**
- **預ける / 受けとめる / 受けとる / 守る**
- **時間 / 心 / ふだん / 暮らし**
- **家族の声 / あなたの声 / いつもの声**

### 1.5 ブランドが避ける概念

- 効果の断定 (治す・改善する・進行を遅らせる)
- 監視・管理・モニタリング・スコアリング (家族向け表現として)
- 罪悪感の喚起 (「あなたが見ていないと…」)
- ハイテク誇示 (「最先端AI」「業界初」「医療レベル」)

## 2. トーン&マナー

### 2.1 基本トーン

DELNE の基本トーンは **「やさしく、丁寧に、けれど主体的に」**。

| 軸 | DELNE のポジション |
|---|---|
| 距離感 | **近すぎず・遠すぎず**。介護家族の「相談できる相手」程度の距離 |
| 温度感 | **あたたかい**。ただし過剰に同情的ではない |
| 主観性 | **抑制的**。私たちの感情ではなく、ご家族の状況を主語にする |
| 専門性 | **奥行きはあるが、表に出さない**。難語・略語を避け、平易に語る |
| 提案性 | **押し付けない**。「使ってください」より「お役に立てるかもしれません」 |

### 2.2 ターゲット別のトーン使い分け

3ペルソナに応じて主トーンを微調整する。

| ペルソナ | 主トーン | 留意点 |
|---|---|---|
| **① 直接介護されている方** | あたたかい・共感的・安心感 | 罪悪感を煽らない。介護そのものを否定しない |
| **② 経営者・人事担当者** | 落ち着き・実務的・信頼感 | 数字・ROI・法改正を扱うが、人間の課題が起点であることを忘れない |
| **③ 遠方在住で直接介護が難しい方** | 後ろめたさを受けとめる・前向き | 「離れていてもできること」を肯定的に伝える |

### 2.3 do / don't の対比

| do (DELNE らしい) | don't (DELNE らしくない) |
|---|---|
| 「優しく話せず自分を責めているあなたへ」 | 「介護で疲れていませんか?(同情的すぎる)」 |
| 「24時間365日、丁寧に応答します」 | 「24時間休まず働く高性能AI!」 |
| 「医療法人社団 松栄会 監修の専門知見」 | 「医療レベルのAIケア」 |
| 「ふだんと違う会話の様子をお知らせ」 | 「異常を検知!すぐに対応を」 |
| 「あなたの声で、お電話にお応えします」 | 「あなたの声をディープラーニング技術で完全再現」 |

### 2.4 感情の取り扱い

- **共感は描写する。煽らない。** 「眠れない夜があります」は描写、「あなたは限界です」は煽り
- **解決は約束しない。並走を約束する。** 「介護を解決」ではなく「介護と仕事の両立を支援」
- **明るさは無理に作らない。** 重さを認めた上で、その隣に小さな選択肢を置く

## 3. 人称・文体

### 3.1 一人称

| 場面 | 推奨 | 備考 |
|---|---|---|
| 製品・サービスを指す | **「DELNE」** | カタカナ表記。ロゴ周辺以外は通常テキストでOK |
| 運営主体としての発信 | **「私たち」** | 「弊社」は営業資料・正式書面のみ。SNS等では避ける |
| 機能の主体 | **「DELNE が / AI が」** | 「弊AI」「弊サービス」等の擬人化された自称は使わない |
| 企業正式名 | **「株式会社オノゴロ」** | プレス・契約書類等で使用 |

### 3.2 二人称

| 対象 | 推奨 | NG |
|---|---|---|
| 介護家族 (ペルソナ①③) | **「あなた」** | 「お客様」「ユーザー」(距離感が遠い) |
| 企業担当者 (ペルソナ②) | **「貴社」「ご担当者様」** | 「あなた」(個別営業文脈以外では距離感不自然) |
| 認知症のある方 (本人を直接指す場面はまれ) | **「ご家族」「ご利用者ご本人」** | 「患者」「対象者」「相手」 |

### 3.3 第三者の呼称

`compliance.md` 4節と整合させる。

| 対象 | 推奨 | NG |
|---|---|---|
| 認知症の本人 | **「認知症のある方」「認知症のご家族」** | 「認知症患者」「ボケた人」 |
| 軽度認知障害 | **「MCI (軽度認知障害) の方」** | 「予備軍」(単独使用) |
| 介護する家族 | **「ご家族」「介護されているご家族」「ビジネスケアラー」** (文脈に応じて) | 「介護者」(冷たい) |
| 行動症状 | **「ふだんと違う様子」「行動・心理症状 (BPSD)」** | 「問題行動」「徘徊」 |

### 3.4 文体

- **基本: です・ます調** (LP・SNS・プレス・営業資料すべて)
- **例外:**
  - キャッチコピー・見出しは体言止め・倒置・口語OK (「でんわにデルネ」)
  - 社内文書・技術文書は **である調可** (本ガイド自身もである調)
  - 引用・お客様の声は原文ママ (改変しない)
- **一文の長さ**: 原則 **60字以内**。介護で時間が無い読者を想定し、短く区切る
- **漢字とひらがなのバランス**: やさしさを出すため、難しい漢字はひらがな化を検討
  - 例: 「致します → いたします」「下さい → ください」「頂く → いただく」
  - ただし「介護」「認知症」等のドメイン語は漢字維持

### 3.5 句読点・記号

- **句読点**: 全角「、。」を使用。半角カンマ「,」「.」は使わない (英文・コード除く)
- **引用符**: 「 」(かぎ括弧) を基本に。サブ引用は『 』
- **強調**: **太字** を基本。下線・色文字は多用しない (読み疲れ防止)
- **感嘆符**: 原則使わない (誇張トーンになる)。SNS等で例外的に「。」「…」で余韻を作る
- **絵文字**: SNSのみ可。LP・プレス・営業資料では原則使わない。使う場合も💊🏥等の医療系・🔥💯等の強調系は避ける

### 3.6 数字・単位

- **数字**: アラビア数字 (半角)。「ひとり」「ふたり」など語感を優先する場合のみ漢字
- **単位**: 「¥7,000/月」「24時間365日」「3ステップ」のように、視認性の高い書式
- **「約」**: 推計値・概数には必ず「約」を付ける (景表法対応)

## 4. キャラクター方針

### 4.1 擬人化の方針

DELNE は **擬人化キャラクターを公式には設定しない**。

理由:
- 「家族の声を再現する」が本質であり、別人格のマスコットを置くと本質と矛盾する
- 認知症の方が誤認するリスク (「キャラと話している」と思わせない)
- ブランドの中心は **介護家族その人の声** であり、DELNE はその透明な媒介者である

### 4.2 「DELNE が話す」場合のペルソナ

SNS発信・カスタマーサポート文面等、DELNE が直接語る場面のペルソナ:

| 属性 | 設定 |
|---|---|
| 性別感 | **中性的** (どちらにも寄せない) |
| 年齢感 | **30〜40代相当**。落ち着いていて、ご家族の話を聞き慣れている |
| 立場 | **隣にいる相談役**。指導者でも友達でもない |
| 一人称 | **「DELNE」「私たち」** (「ぼく」「わたし」等の人格的一人称は使わない) |

### 4.3 アイコン的表現の余地

将来的にイラストモチーフを使う場合、以下の方針:
- **電話・受話器** をモチーフとした抽象アイコンは可
- 顔・人体のキャラクター造形は **避ける** (家族の顔を限定しないため)
- 必要なら **手・声の波形・電話線・窓辺** 等のシンボリック表現を選ぶ

## 5. カラーパレット (暫定)

> 🔧 本章は **暫定値**。確定にはデザイナーレビュー・印刷物校正・アクセシビリティテストを要する。
> 確定までの間、各成果物では本暫定値を運用基準とする。

### 5.1 設計思想

- 医療らしさより **「暮らしの中の安心感」** を優先
- 過度な暖色 (赤・橙) は介護不安を喚起しうるため避ける
- 過度な冷色 (深いネイビー・グレー) は医療機器的になるため避ける
- 主軸は **やわらかいブルー〜グリーン系** + あたたかいアクセント

### 5.2 暫定パレット

| 役割 | 色名 | HEX (暫定) | 用途 |
|---|---|---|---|
| **プライマリ** | DELNE Blue | `#3A7BC8` | ロゴ・主要CTA・キーアクセント |
| **セカンダリ** | Soft Sky | `#A8C8E8` | 背景パターン・装飾 |
| **アクセント** | Warm Coral | `#F2A48C` | 強調・温かさを添えるポイント |
| **ベース (明)** | Off White | `#F8F6F2` | 背景・余白 |
| **ベース (暗)** | Ink Gray | `#2C3540` | 本文テキスト |
| **サブテキスト** | Mid Gray | `#6B7480` | 補足・キャプション |

### 5.3 使用比率の目安

```
ベース (明):  60%   余白を多く、息のつける紙面
プライマリ:   25%   ロゴ・CTA・キービジュアル
サブテキスト: 10%   本文・補助
アクセント:    5%   ここぞの一点
```

### 5.4 避けるべき配色

- 赤一色を主軸にした警告・緊急トーン
- 蛍光色 (ネオングリーン・ホットピンク等)
- 高彩度の原色を複数並べる多色構成 (キッズトーンになる)

### 5.5 アクセシビリティ

- 本文テキストと背景のコントラスト比 **4.5:1 以上** (WCAG AA)
- CTA ボタンは背景と **3:1 以上**
- 色だけで情報を区別しない (アイコン・テキスト併用)

## 6. タイポグラフィ (暫定)

> 🔧 本章も **暫定値**。LP実装・印刷物制作と合わせて検証する。

### 6.1 推奨フォント

| 言語 | フォント | 代替 |
|---|---|---|
| 日本語 | **Noto Sans JP** | ヒラギノ角ゴ Pro / 游ゴシック |
| 欧文 | **Inter** | Helvetica / Arial |
| ロゴ用 | (要デザイン確定) | - |

理由: 可読性が高く、Webフォントで無償提供されており、医療らしさより日常の親しみを表現できる。

### 6.2 ウェイト運用

- 本文: **Regular (400)**
- 太字強調: **Bold (700)**
- 見出し大: **Bold (700)** または **Medium (500)** + サイズ大
- **細字 (Light/Thin)** はキービジュアル以外で多用しない (病院的に見える)

### 6.3 サイズ階層 (Web基準・暫定)

| 階層 | サイズ | 用途 |
|---|---|---|
| Hero | 48〜64px | LPトップ・キャッチコピー |
| H1 | 32〜40px | セクション見出し |
| H2 | 24〜28px | サブ見出し |
| H3 | 18〜20px | 小見出し |
| Body | 16〜18px | 本文 (高齢ユーザー考慮で **16px下限**) |
| Caption | 14px | キャプション・注釈 |

### 6.4 行間・字間

- 本文 line-height: **1.7〜1.9** (日本語可読性のため広め)
- 見出し line-height: **1.3〜1.4**
- 文字詰め (letter-spacing): 本文 0、見出し +0.02em 程度

## 7. ロゴ運用 (暫定)

> 🔧 ロゴデザイン本体の確定は別タスク。本章は **運用ルールの枠組み** を先に定める。

### 7.1 基本ルール

- 必ず公式データ (SVG/PNG) を使用。文字起こしや再描画は禁止
- 比率は変更しない (縦横比固定)
- 色は本ガイドのプライマリカラー、または白抜き・黒のみ
- 影・縁取り・グラデーション等の装飾は付けない

### 7.2 アイソレーション (余白)

- ロゴの周囲には、**ロゴ高さの 1/2 以上の余白** を確保する
- 余白内に他要素 (文字・画像) を配置しない

### 7.3 最小サイズ

- Web: **横幅 80px 以上**
- 印刷: **横幅 20mm 以上**
- 視認性が確保できないサイズでは使用しない

### 7.4 禁止例

- 色の変更 (パレット外)
- 回転・歪み・反転
- 部分的な切り取り・トリミング
- 透過率を下げて背景化する使用 (CTAボタン背面のロゴ等)
- 写真の上に直接重ねる (コントラストが確保できない場合)

### 7.5 共起表記

- 「医療法人社団 松栄会 監修」「ひろしまAIサンドボックス採択」等のクレジットを併記する場合、
  ロゴと **同等以上の余白** を取ってから配置する

## 8. ビジュアル方針

### 8.1 写真のトーン

- **やわらかい自然光**。フラッシュ・ハイコントラストは避ける
- **手元・後ろ姿・窓辺・電話のある暮らし** など、顔の特定に依存しない構図を優先
- 認知症のある方を直接被写体にする場合は、**ご本人とご家族の同意取得済** であることを文書管理 (本ガイドの責任範囲外。撮影時運用で対応)
- **ステレオタイプを避ける**: 「悲しむ高齢者」「うつむく介護者」一辺倒にしない。穏やかな日常の瞬間も使う

### 8.2 イラストのトーン

- 線は **やわらかく・手描き感のあるもの**
- 配色はパレットに準拠
- 4.3 にあるとおり **顔・人体の具体的造形は避ける**
- アイコンは **線画 (line) と面 (solid) を混在させない**。1コンテンツ内で統一

### 8.3 グラフィック要素

- 角丸を基本に (角ばった矩形・三角は機械的に見える)
- 余白を多くとる (情報過多にしない)
- 円・楕円・波形など **やわらかい幾何形** を使う

### 8.4 動画・音声素材

- BGM は **静かなピアノ・アコースティック・環境音** を基本
- 急展開・ジャンプカット・派手なテロップは避ける
- 「家族の声を再現する」サービスの性格上、**実在の人物音声サンプル** の取り扱いには細心の注意 (本人同意・改変表示)

## 10. チェックリスト (制作時セルフチェック)

成果物完成後、以下を確認する。

### 10.1 法令・配慮 (compliance.md ベース)

- [ ] 医療効果の断定表現は無い (予防・改善・進行抑制 等)
- [ ] 優良誤認になる表現は無い (必ず・100%・日本一 等)
- [ ] 認知症のある方を尊重した呼称 (「認知症の方」)
- [ ] 家族介護者を責めるトーンになっていない
- [ ] 必須記載事項 (数値の出典・調査対象・期間) が揃っている

### 10.2 トーン (本ガイド 2章)

- [ ] 共感は描写されており、煽りになっていない
- [ ] 解決を約束していない (並走を約束している)
- [ ] 一文 60字以内を意識した区切りができている
- [ ] ターゲット別のトーンが適切に選ばれている

### 10.3 人称・文体 (本ガイド 3章)

- [ ] 一人称・二人称が統一されている
- [ ] です・ます調が基本になっている
- [ ] 「ユーザー」「弊AI」等のNG語が無い
- [ ] 漢字・ひらがなのバランスが過度に硬くない

### 10.4 ビジュアル (本ガイド 5〜8章)

- [ ] カラーパレット内の色のみ使用している
- [ ] フォントが推奨フォントになっている
- [ ] ロゴの余白・最小サイズが守られている
- [ ] 写真・イラストが本ガイドのトーンに沿っている
