# `_threads` ディレクトリ

複数の Claude Code on the web セッション (スレッド) を並列運用する際の **状態共有・調整用** ディレクトリ。
各セッションは独立してメモリを持たないため、git リポジトリを介して状態を共有する。

## 構成

```
_threads/
  README.md            本ファイル (運用ルール)
  consulting-mode.md   壁打ちモード規約 (全スレッド共通の振る舞いルール)
  active.md            全スレッドの現状サマリ (各スレッドが自セクションを更新)
  <thread-name>/
    brief.md           当該スレッドの目的・スコープ・参照ファイル (基本静的)
    log.md             意思決定・成果物リンクの記録 (追記型)
```

## 既定モード: 壁打ち

全スレッドの既定モードは **壁打ちモード** (ユーザー=シニアコンサルタント、Claude=アナリスト兼スパーリングパートナー)。
詳細は [`_threads/consulting-mode.md`](./consulting-mode.md) を参照。
セッション開始時に必読のこと。

## スレッド一覧

| 表示名 | ディレクトリ | 種別 | 状態 |
|---|---|---|---|
| DELNE-marketing 統合企画 | `integration` | 長期維持 | アクティブ |
| LP表現改訂 | `lp-revision` | 短期 | 待機 |
| ブランドガイドライン作成 | `brand-guide` | 短期 | 待機 |
| SNS運用企画 | `sns-strategy` | 短期 | 待機 |
| プレスリリース企画 | `press-strategy` | 短期 | 待機 |

新規スレッドを立ち上げる場合は本テーブルに追記し、`<name>/brief.md` を作成する。
新規スレッドの立ち上げ判断は **統合企画スレッド** が行う。

## 運用ルール

### セッション開始時

1. `git checkout main && git pull origin main` でリポジトリを最新化
2. `_threads/active.md` を読み、他スレッドの直近状態と依頼事項を把握
3. 自スレッドの `brief.md` と `log.md` を読み、文脈を確認

### セッション中

- 重要判断・成果物作成のたびに自スレッドの `log.md` に追記
- 他スレッドへの依頼事項がある場合は、自スレッドセクション内の「他スレッドへの依頼」欄に書く
- スコープ外の判断が必要になったら、統合企画スレッドへ上申 (active.md 経由)

### セッション終了時

1. 自スレッドの `log.md` に最終的な判断・成果物リンクを追記
2. `_threads/active.md` の自スレッドセクションを更新 (状態・直近の論点・次回目標)
3. 成果物を該当ディレクトリ (`projects/delne/marketing/<category>/`) に配置
4. `git add -A && git commit -m "<type>: <要約>"`
5. `git push origin main` (失敗時は `git pull --rebase origin main` してリトライ)

## コンフリクト回避ルール

- **書き込み排他**: 各スレッドが書く先は **自分の `log.md` のみ** → コンフリクト無し
- **共有書き込み**: `active.md` は全スレッド共有。**スレッドごとのセクションのみ** 局所更新。他セクションには触れない
- **特権ファイルの更新権限**: 以下は **統合企画スレッドのみ** が更新可能:
  - `CLAUDE.md`
  - `projects/delne/product/summary.md`
  - `projects/delne/marketing/strategy/compliance.md`
  - `_threads/README.md`
  - 各スレッドの `brief.md` (初期設定後の変更)
- 他スレッドから上記ファイルへの変更提案がある場合は、`active.md` 経由で統合企画に上申

## 完了したスレッドの扱い

- 主要成果物が完成したら、自スレッドの `log.md` に「**完了**」マーク
- `active.md` の状態を「完了」に変更
- ディレクトリ自体は残す (履歴参照用)
- 統合企画スレッドが必要に応じてアーカイブ判断を行う

## トラブルシューティング

### push が拒否された (他スレッドが先に push)

```sh
git pull --rebase origin main
# コンフリクトが出なければそのまま push
git push origin main
```

### コンフリクトが出た

- 自分の `log.md` のコンフリクト → 自分の変更を優先
- `active.md` の **自セクション** のコンフリクト → 自分の変更を優先
- `active.md` の **他セクション** のコンフリクト → 他スレッドの変更を採用 (自分は他セクションに触れていないはず)
- 上記以外でコンフリクト → 統合企画スレッドへ上申
